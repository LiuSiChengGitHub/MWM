## 这个课题整体在干什么

这个课题做的是“医疗世界模型”相关研究。  
如果用最基础的话来说，它不是只想做一个“输入病人信息，输出一个标签”的普通分类模型，而是想做一个更像“患者状态模拟器”的系统。

可以把它理解成三层：

1. 普通预测任务
   例如预测病人会不会出现低血压、会不会大出血、会不会出现术后并发症。
2. 时间序列建模
   不只是看某一个时间点，而是看病人的状态如何随时间变化。
3. 世界模型
   不只是判断“会不会发生”，还希望理解：
   - 病人现在处于什么状态
   - 如果采取不同干预，未来状态会怎么变化
   - 哪种干预可能带来更好的结局

老师当前给出的围术期方向重点是：

- 利用术前基础特征
- 结合术中波形和其他术中数据
- 预测术中大出血
- 预测术中低血压
- 预测术后并发症

所以从整体上看，这个课题想做的是：  
**基于临床多模态数据，学习患者状态在医疗干预下如何演化，并进一步服务于预测、模拟和决策支持。**

## MIMIC-IV 数据集包含什么

我目前下载并确认好的数据主要在 `../datasets_MWM/` 下，包含：

- `mimic-iv-3.1`
- `mimic-iv-ed-2.2`
- `mimic-iv-note_2.2`

其中我当前重点理解的是前三类模块：

### 1. `hosp`

这是住院级结构化数据，主要包括：

- 患者信息
- 住院信息
- 诊断编码
- 化验结果
- 处方记录
- 转科记录等

它比较适合做住院层面的数据理解和标签构造。

### 2. `icu`

这是 ICU 相关的数据，也是时序信息最丰富的一部分，主要包括：

- ICU stay 信息
- 监护记录
- 输入输出事件
- 过程性事件

这一部分很适合后续做时间序列任务，也是我觉得最接近“患者状态演化”的数据来源之一。

### 3. `ed`

这是急诊数据，主要包括：

- 急诊停留记录
- 分诊信息
- 生命体征
- 急诊诊断

它适合做早期风险分层或急诊到住院/重症的转归分析。

### 4. `note`

这是文本类数据，目前我本地看到它还是 zip 压缩包，后面如果涉及文本信息或多模态扩展，可能会进一步使用。

## 这个数据集的结构怎么理解

我目前先记住了 5 个最核心的字段：

- `subject_id`：患者级 ID
- `hadm_id`：一次住院的 ID
- `stay_id`：一次 ICU 或 ED 停留的 ID
- `itemid`：监测项或化验项的字典编号
- `icd_code`：诊断或操作编码

其中最重要的关系可以先理解成：

- `patients -> admissions -> icustays`
- `patients -> edstays`

也就是说：

- 一个患者可以有多次住院
- 一次住院可能没有 ICU，也可能进入 ICU
- 急诊停留是另一条比较重要的就诊路径

另外，很多真正可用于建模的内容并不在“主表”里，而是在事件表里，比如：

- `labevents`
- `chartevents`
- `inputevents`

这些表通常按时间记录病人的检验、监护或治疗事件，所以后续如果要做时序任务，这些表会比较关键。

还有两类字典表也很重要：

- `d_labitems`
  - 用来解释 `labevents` 里的 `itemid`
- `d_items`
  - 用来解释 `chartevents`、`inputevents` 等表里的 `itemid`

所以对这个数据集的结构理解，不能只停留在“有很多 CSV 文件”，而是要理解：

- 哪些是患者层级
- 哪些是住院层级
- 哪些是 stay 层级
- 哪些是时间事件表
- 哪些是字典表

## 这个数据集怎么用

如果从数据工作的角度讲，这个数据集大概是这样用的：

1. 找入口表
   - 比如住院任务从 `admissions` 开始
   - ICU 任务从 `icustays` 开始
   - 急诊任务从 `edstays` 开始
2. 选研究对象
   - 例如只看 ICU 病人，或者只看急诊后入院病人
3. 选时间窗
   - 比如取入 ICU 后前 6 小时或前 24 小时数据
4. 定义 label
   - 例如住院死亡、低血压风险、是否转 ICU
5. 抽取特征
   - 从人口学、诊断、化验、生命体征、治疗事件中提取特征
6. 导出训练数据
   - 把原始多表数据整理成后续模型可以直接使用的样本

对我现在来说，重点不是马上做复杂建模，而是先学会两件事：

- 读懂数据
- 把原始表整理成后续能用的数据样本
